GIT Installation Setup

1.      Setup an account on https://www.github.com with

a.      Username: CIS-XXXX (eg. CIS-pnbokka)

b.     Email: your dhs email address

c.      Enable 2FA i.e Two Factor Authentication

Login to your GitHub account then go to Settings à Security à turn on Two-factor authentication

 

 

 

 

2.      Download TortoiseGit (TortoiseGit-1.8.15.0-64bit.msi)from Serena.

3.      Download Git for Windows (Git-2.5.0-64-bit.exe).

4.      Download Git Lfs.

Installation

Logoff and log back in to GFE with workstation admin account.

TortoiseGit installation

Accept all defaults

1.      Click Next on Welcome page.

2.      Click Next on Information page.

3.      Accept defaults on Custom Setup and click Next.

4.      Click Install on the next page and

5.      Complete setup wizard and Finish.

Git for Windows Installation

1.      Starting installation opens up Git Setup Wizard and click Next.

 

2.      Click Next on Information page.

 

3.      Select Destination Location Step is important. Git should be installed in C:\Program Files\Git path only. You have to log in as workstation admin on your laptop to install in “Program Files” folder on your GFE otherwise need open a ticket with helpdesk.

Accept the defaults and click Next.

 

 

 

 

 

 

Setup is now complete.

 

Cloning a Remote Repository Locally

 

Open Explorer à Right click  à Click Git Clone

 

 

1.      Copy SSH URL for GitHub repository to clone locally or use the default repository configured earlier.

Select Load Putty Key and Select the Private Key saved earlier.

Cloning a remote repository locally is complete.

5.      Commit the file to local repository using Git commit.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

6.      Sync the local repository with the Remote repository and push files to remote branch.

Right click on the local repository and in the  dialog click Git Sync click Push button.

 

Note: As Firewall is not open between GFE connected to VPN and GitHub.com, VPN needs to be disconnected before the Push/Pull/Git Clone operations.

But this step is not required when connecting to GitHub Test Environment and Git Hub Enterprise environments as they reside within the network.
